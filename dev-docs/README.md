# Developer Docs

This directory contains maintainer and implementation documentation. These files are not part of the public author/contributor docs surface.

Use this directory for:

- Product & architecture reference — the current (v1.0) state and the single starting point for new work: `PRD.md`

`PRD.md` is self-contained. It consolidates what were previously separate per-milestone build plans and architecture decision records, so the durable knowledge (the Standard, decisions and their rejected alternatives, resolved micro-decisions, architecture, and open items) lives in one place.

Public quiz author and Catalog contributor docs live in `docs/`.
